Get Started with ServiceStack

Checkout the different ways to get started with ServiceStack. Project templates, Walk-throughts, Documentation and Videos, choose the best one for you.

Get the .NET SDK

If you haven't already got the .NET SDK installed, it's recommended you start with the latest .NET SDK 5.0 or higher.

ServiceStack runs on both the older .NET Core LTS versions as well as the Windows-only .NET Framework but to get the best experience, use the latest .NET SDK 5.0 or higher.

Download

Download a template

ServiceStack has many templated projects so you can get started quickly on a new project. They are available to download as a zip here as well as a link to their source on GitHub.

These templates can also create projects with ServiceStack dotnet tooling which can be installed using the command dotnet tool install -g x. The x tool not only has project templates, but supports many other great workflows as well.

Templates are designed with developer workflows in mind, specific to each template. Building on existing OSS tooling like Vue CLI, and useful NPM scripts, you'll get everything you need to start integrating with your ServiceStack server quickly right from the beginning.

Open and Run

The templates all have a MyApp.sln file which you can open with your favorite IDE.

The Recommended project structure separating the Host, Service Interface, Service Model and Tests is a common part of the ServiceStack templates, this ensures ServiceStack solutions starts off from an optimal logical project layout, laying the foundation for growing into a more maintainable, cohesive and reusable code-base.

How does it work?

Now that your new project is running, letâ€™s have a look at what we have. The template comes with a single web service route which comes from the request DTO (Data Transfer Object) which is located in the Hello.cs file.

The Route attribute is specifying what path /hello/{Name} where {Name} binds its value to the public string property of Name.

The MyServices class is a ServiceStack service with a method utilizing the Hello and HelloResponse DTOs. The name of the method Any means the server will run this method for any of the valid HTTP Verbs. Service methods are where you control what returns from your service.

Accessing the route using your browser via the /hello/world route, you will see a snapshot of the Result in a friendly HTML response format. To change the return format to JSON, simply add ?format=json to the end of the URL.
