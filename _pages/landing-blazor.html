Everything you need to develop Blazor WASM Apps

ServiceStack Blazor WASM templates improve on standard Blazor WASM development model by providing end to end typed APIs that integrate with your services.

Static Page Generation

Templates have a static site generation solution to provide instant loading on your content. Perfect for great UX and SEO.

GitHub Actions

Deploy straight from GitHub Actions with built in workflows. Supports usage of CDN for static site and simple Linux host for .NET.

Blazor Components

Templates come with ServiceStack.Blazor components that provide validation and error handling out of the box.

Markdown Support

Create content pages with ease using Markdown. Pre-rendered for a complete static site generation based solution.

Tailwind Support

Blazor WASM components & templates come with a TailwindCSS version to make great looking pages with TailwindUI components.

Shared DTOs

Shared Request & Response DTOs create a typed end-to-end API that is instantly usable from Blazor WASM.

Use TailwindUI components with Blazor WASM

As well as the standard Bootstrap based template, we provide a `blazor-tailwind` template so you can get started quickly taking advantage of TailwindUI components. ServiceStack.Blazor components also support TailwindCSS so you can work on a clean Tailwind-first project without spending time manually removing bootstrap related elements.

TailwindCSS Live Reload works hand in hand with Blazor WASM hot reload to provide an extremely productive development loop with a great looking UI.

Checkout a live hosted demo of the Blazor Tailwind template to see the Booking CRUD, Todos, Counter and Weather example pages in action. These pages provide concrete examples of how to take advantage of the ServiceStack Blazor development model with reusable Request & Response DTOs.

Reuse typed APIs in Optimized UIs

Locode's declarative dev model lets you focus on your new App's business requirements where its data model, API capabilities, input validation & multi-user Auth restrictions can be defined simply using annotated C# POCOs.
This provides immense value at the start of the development cycle where functional prototypes can be quickly iterated to gather business requirements

Once requirements have been solidified, the typed AutoQuery APIs can easily be reused to develop custom UIs to optimize important workflows.

Talent Blazor is a new App showcasing an example of this where its entire back-office functionality can be managed through Locode whilst an optimized Blazor WASM App is created to optimize its unique workflow requirements which also benefits from the superior productive dev model of its Typed APIs
